# Adie Vibe Coding Workflow Skill — 使用说明

## 这个 Skill 能做什么

一个完整的项目开发引导流程。从口喷需求开始，到文档生成、MVP 开发、部署上线，全程交互式引导。

**一句话：你不需要记住流程，Agent 会带着你走，你随时问"下一步干啥"就行。**

---

## 使用方式

### 场景一：开发前（需求收集 → 文档生成 → 开发计划）

> 目标：把你脑子里模糊的想法，变成 AI 能执行的文档和计划。

**步骤：**

1. **口喷需求**
   找 GPT（或其他 AI）把你对产品的需求想法说出来，让它生成 1-4 份 HTML 问卷（合计不超过 200 题）。

2. **填写问卷 + 对话**
   - 打开问卷认真填写，不懂的题去问 GPT，把回答抄到问卷底部补充区
   - 同时保留你和 GPT 的完整对话链接

3. **把材料喂给 Agent**
   把填好的问卷 + GPT 对话链接一起给你的 Agent（Claude Code / Codex / Cursor 等）。

4. **Agent 自动生成文档**
   Agent 会基于这些材料按 Skill 流程自动生成：
   - `FINAL_REQUIREMENTS_SUMMARY.md` — 需求决策摘要（你审核确认）
   - `PRD.md` — 产品需求文档
   - `UI.md` — 前端 UI 文档
   - `TECH.md` — 技术文档
   - `AGENTS.md` + `CLAUDE.md` — 项目级 Agent 持久化指令
   - `PLAN.md` — MVP 开发计划

5. **审核确认 → 开始开发**
   你看完文档和计划没问题后，Agent 开始按 PLAN.md 逐个 Task 执行。

---

### 场景二：开发中（进度查询 + 需求变更 + 文档维护）

> 目标：开发过程中保持方向清晰，需求变了文档同步更新，不脱节。

**随时可用的交互指令：**

| 你说 | Agent 做 |
|------|---------|
| "我们现在到哪了" / "下一步干啥" | 告诉你当前阶段、下一步要做什么 |
| "更新需求：XXX" | 更新 PRD/UI/TECH 对应文档 |
| "review 一下文档" | 检查所有文档是否和最新需求一致 |
| "安全检查" | 检查 API Key、加密、认证相关内容 |
| "生成部署清单" | 输出部署前的检查清单 |
| "跳到阶段X" | 跳转并检查前置条件 |

**关键纪律：**
- 每轮改动完，Agent 会提醒你是否要 `git commit`
- PLAN.md 是活的计划书，完成/废弃/新增都会实时更新状态
- 需求变了文档必须跟着变，Agent 会主动提醒你同步
